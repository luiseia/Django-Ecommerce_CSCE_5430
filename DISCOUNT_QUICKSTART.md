## 📋 商家的折扣管理界面

### 折扣列表页

- 快速操作：查看详情、编辑、删除

### 创建新折扣
**点击 "Create New Discount" 按钮**

填写表单：
```
代码: SUMMER20 (必填，大小写自动转换)
说明: 夏季促销 $20 优惠 (可选)
折扣类型: 
  - Percentage (%) → 输入 20 = 20% 折扣
  - Fixed Amount ($) → 输入 20 = $20 折扣
折扣金额: 20
最低购买金额: 100 (订单总金额要 ≥ $100)
最低商品价格: 100 (单商品价格要 ≥ $100)
最大使用次数: 留空 = 无限 (或输入具体数字)
有效期: 
  - 开始: 2026-03-29 10:00
  - 结束: 2027-03-29 23:59 (可选，留空=永不过期)
适用商品: 选择想要的商品，留空=全部商品
启用: ✓ 勾选 (确保折扣可用)
```

## 🔍 常见问题

**Q: 为什么购物者看不到我的折扣？**
A: 检查：
- [ ] 折扣是否启用 (Active 状态)
- [ ] 购物车金额是否满足最低金额要求
- [ ] 商品是否在允许列表中
- [ ] 折扣是否超出使用次数限制

**Q: 可以为某个商品设置多个折扣吗？**
A: 可以，但购物者一次只能使用一个折扣代码，会自动选择第一个有效的

**Q: 如何禁用折扣但保留数据？**
A: 在编辑页取消勾选 "Active" 复选框，不需要删除

**Q: 可以改变折扣代码吗？**
A: 不行，代码是固定的。需要改代码的话只能删除重建

## 🛠️ 技术细节

### 折扣代码验证流程
```
输入代码 → 检查存在性 → 检查激活状态 → 验证有效期 
→ 验证使用次数 → 检查最低金额 → 计算折扣 → 应用
```

### 数据库字段
```python
DiscountCode 模型:
- code: 折扣代码 (唯一，同商家内)
- merchant: 所属商家
- discount_type: 百分比/固定金额
- discount_value: 折扣值
- min_purchase_amount: 最低订单额
- min_product_price: 最低商品价格
- products: 适用商品列表 (ManyToMany)
- max_uses: 最大使用次数
- current_uses: 当前使用次数
- valid_from/valid_until: 有效期
- is_active: 是否激活
```

## 🎓 进阶用法

### 创建分层折扣
```
SAVE10: $50+ 减 $10
SAVE20: $100+ 减 $20
SAVE50: $500+ 减 $50
```
购物者会自动适用等级最高的有效折扣

### 限时促销
设置 `valid_until` 字段为活动结束时间，系统会自动禁用

### 品类折扣
在 "Applicable Products" 只选择特定品类的商品
